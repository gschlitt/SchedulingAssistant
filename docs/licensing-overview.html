TermPoint — Licensing & Trial Overview

This document describes how TermPoint handles licensing and the free trial period. It is intended for IT staff who may be asked to help with license installation, and as a reference for the development team.

1. How Licensing Works

TermPoint uses a department-level license, not a per-user license. A single license file covers everyone in the department who points TermPoint at the same shared database folder.

The license file

  • Filename: termpoint.lic
  • Location: the same folder as the department's .tpdb database file
  • Format: a small text file with a human-readable header and a cryptographic signature

Opening the file in a text editor shows something like:

# TermPoint License
# Department: Geography Department
# Issued:     2026-09-01
# Expires:    2027-09-01
#
# Do not edit below this line.

eyJkZXBhcnRtZW50IjoiR2VvZ3Jh...   (encoded payload)

uKZZWk7ct1LxQ39DqhRB+UlYPmP...   (digital signature)

The encoded sections are digitally signed. Any modification to the file (even changing a single character) invalidates it. TermPoint verifies the signature on every launch — no internet connection is required.

Installing a license

Place the termpoint.lic file (received after purchase) in the same folder as the department's database. That's it — TermPoint finds it automatically on the next launch.

Multiple databases: If the department maintains databases in separate folders (e.g. one per subject area), a copy of termpoint.lic is needed in each folder.

2. The 30-Day Trial

When TermPoint launches without a license file, it offers a 30-day free trial with full editing access.

How the trial clock works

  • On first launch, TermPoint creates a small file called trial.json in the user's local application data folder: %APPDATA%\TermPoint\trial.json
  • This file records the date of first launch. The trial period is calculated from that date.
  • The trial clock is per machine — each computer that runs TermPoint has its own independent trial.
  • The trial clock starts on the very first launch, even if a license file is present. This is by design (see Trust Model below).

What happens when the trial expires

After 30 days without a license file, TermPoint enters read-only mode. The user can still open and view the schedule, but cannot make edits. A banner explains the situation and provides a link to purchase a license.

3. Access States

License file Trial status Result
Valid (not expired) (any) Full access — editing enabled
Permanent (no expiry) (any) Full access — editing enabled
Expired (any) Read-only — banner shown with renewal link
Missing or corrupt Within 30 days Full access — trial active, days remaining shown
Missing or corrupt Past 30 days Read-only — trial ended, purchase link shown

4. Common Scenarios

New department setup

Install TermPoint, create a database, start working. The 30-day trial begins immediately. Purchase a license before it expires and place termpoint.lic in the database folder.

New staff member joins

The new person installs TermPoint on their machine and points it at the existing shared database folder. If a valid termpoint.lic is already there, they get full access immediately — no separate license needed.

License renewal

When a license expires, TermPoint enters read-only mode on the next launch. Replace the old termpoint.lic with the renewed one. Full access is restored on the next launch.

License file accidentally deleted

If someone deletes termpoint.lic from the database folder, TermPoint falls back to the trial. Since the trial clock started on the original first launch (typically months earlier), the trial will already be expired and the app goes to read-only. Restoring the license file from backup (or requesting a replacement) fixes it.

5. Trust Model

TermPoint's licensing is designed for university departments — a cooperative environment where the goal is honest compliance, not adversarial copy protection.

The license file is a plain file on a shared drive. There is nothing preventing someone from copying it to another department's folder. The system trusts that departments will purchase their own licenses rather than share them. This is a deliberate trade-off: simple, no-hassle licensing that works reliably in diverse university IT environments, with no phone-home, no activation servers, and no internet requirement.

Similarly, a technically inclined user could reset their trial by deleting %APPDATA%\TermPoint\trial.json, but this is not a concern for the target audience.

6. Troubleshooting

TermPoint says "Trial ended" but we have a license

  • Confirm that termpoint.lic is in the same folder as the .tpdb database file.
  • Open the .lic file in a text editor and check the # Expires: line — is it in the past?
  • Make sure the file hasn't been modified. Even a stray character will invalidate the signature.

Different users see different license states

Each machine has its own trial clock. If one person installed months ago and another installed recently, they may have different trial states. This only matters when there is no valid license file in the database folder.

Multiple database folders

Each folder needs its own copy of termpoint.lic. If one folder has the license and another doesn't, the user will see full access for one database and trial/read-only for the other.
