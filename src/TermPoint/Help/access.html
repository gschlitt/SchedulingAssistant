Using Access Watches

The Access panel helps monitor whether students in a particular program or cohort can actually access all the courses they need. It works by detecting time conflicts among the sections that belong to a group you define — called a "watch." Any conflicts are indicated on Schedule View; it's up to you what you wish to do about them.

If two sections from different courses overlap in time, and both fall within the same watch, TermPoint flags it as a conflict. This tells you that a student who needs both courses cannot attend both.

While filtering to a program or cohort can provide you with quick visual check against conflicts, an access watch is an easy double-check in case you forget or the schedule becomes too visually complex.

User examples

  • A scheduler wants to ensure that the sections which serve their BA program have minimal conflicts. The scheduler tags the courses serving the BA with the tag 'BA' and enrols a watch using that tag. As scheduling proceeds, if two sections serving the BA conflict in time, the user is advised on Schedule View (conflicting meetings are highlighted)
  • Watches can be refined: the scheduler might only be concerned about potential conflicts among upper-level sections serving the BA
  • A scheduler might just want to check access for an ad hoc group of courses (rather than those unified by a tag). The scheduler can easily form such a group and be confident that any conflicts will be flagged.

Access watches examine conflicts across different courses, not within the same course. Two sections of the same course overlapping is normal — a student only takes one.

Where to Find It

The Access panel appears in the filter toolbar above the schedule grid. It shows a collapsed badge that summarizes how many watches are active and how many conflicts exist. Click the badge to expand the watch list and manage your watches.

Creating a Watch

Click + New Watch in the expanded Access panel. A small inline form appears with two mode options:

Tag/Level-based mode

Define the watch by selecting one or more tags and/or levels. This is best when your sections are already tagged by program or cohort (e.g. "Pre-Med", "Honours").

  • Tags (AND logic) — a section must carry all selected tags to be included. For example, selecting both "Pre-Med" and "Year 1" means only Pre-Med sections in first year are watched.
  • Levels (OR logic) — a section matches if its level equals any selected level. For example, selecting "100" and "200" watches sections at either level.
  • You can combine tags and levels: a section must pass both the tag filter AND the level filter.

Course-based mode

Directly select the specific courses whose sections should be watched. Use this when you know the exact courses a student cohort must take, regardless of tags.

A name is auto-generated as you make selections. You can edit it to something more memorable (e.g. "First-year Biology cohort").

Click Create to save the watch. It immediately becomes active and conflicts (if any) appear on the grid.

Reading the Results

Once a watch is created, TermPoint checks for time overlaps among the sections it covers. Results appear in two places:

  • On Schedule View: conflicting sections are highlighted in red, with a line connecting them if they are in different, but overlapping, blocks.
  • Collapsed badge — shows "2 watches, 3 conflicts" at a glance.
  • Expanded watch list — each watch shows its own conflict count and the specific section pairs that overlap (e.g. "MATH 101 A / CHEM 121 B").

Conflicts are recalculated automatically whenever sections are added, edited, or removed.

Managing Watches

Enabling and disabling

Each watch has a checkbox toggle. Disabling a watch keeps it in the list but stops it from generating conflict reports. This is useful if you want to focus on one watch at a time without deleting others.

Deleting a watch

Click the × button beside a watch to remove it permanently. The conflict count updates immediately.

Practical Tips

  • Start with tags. If you've already tagged courses by program (under Scheduling Environment), tag-based watches are the fastest to set up and maintain — new sections automatically inherit their course's tags.
  • Use levels to narrow scope. A "Pre-Med" tag might span 4 years of courses. Adding a level filter (e.g. "100") lets you check just the first-year experience.
  • Create multiple watches. You might have one watch per program or per year-level. Each watch reports its own conflicts independently.
  • Watches are per-semester. Switching semesters loads the watches defined for that semester. If you need the same watch in both Fall and Spring, create it in each.

Access watches only detect conflicts among the sections currently visible in the grid. If you have active filters hiding some sections, those hidden sections are not included in conflict detection.
