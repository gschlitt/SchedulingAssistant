First Launch

When you open TermPoint for the first time, you will be prompted to either open an existing database or create a new one. TermPoint stores all data in a single SQLite file (a .db file) that you keep anywhere on your drive.

  • New database — choose a location and file name. TermPoint will create the file and seed it with default scheduling settings.
  • Open existing — browse to a .db file you already have. Recent files are listed in the Files menu for quick access.

Keep the database file on a network share if multiple administrators need to take turns editing. TermPoint uses a write-lock so only one person edits at a time — others can still open the file in read-only mode.

The Main Layout

Once a database is open you'll see three panels:

  • Section List (left top) — lists every section in the current semester. Click a section to select it on the grid; double-click to open its editor.
  • Workload Panel (left bottom) — summarizes teaching load per instructor.
  • Schedule Grid (right) — the main weekly view, Monday–Friday, 08:30–22:00.

The panels can be detached into floating windows using the ⎘ icon in each panel header. Drag them wherever suits your screen.

Choosing an Academic Year and Semester

The Academic Year dropdown and the Semester checkboxes in the top-right corner of the toolbar control which data is shown. Select one or more semesters to compare them side by side on the grid.

Academic years must be created before semesters can be added to them. Use Academic Years in the toolbar, then add semesters from the semester list inside that flyout.

Escape Close the current flyout panel
