Managing Sections

Sections are the core data entity in TermPoint. A section represents one scheduled offering of a course — its time, room, instructor, and other attributes for a given semester.

Adding a New Section

Click the + Add Section button at the top of the Section List panel. A new section form appears at the top of the list. The editor uses a step-gate sequence to avoid partial data:

  1. Choose a course from the dropdown. Until a course is selected, the Section Code field is disabled.
  2. Enter a section code (e.g. A, 01, ONLINE). The code must be unique within the same course and semester. Press Tab or click away to validate it.
  3. Once the code is confirmed, all other fields unlock: instructor, room, meeting time, section type, tags, and so on.

If you use a naming convention for section codes (e.g. HY1, HY2), select a Section Prefix from the picker that appears after choosing the course. TermPoint will auto-generate the next available code and immediately unlock all fields.

Editing an Existing Section

Double-click any section in the Section List, or double-click its tile on the Schedule Grid. The section expands inline showing the full editor. Only one section can be open for editing at a time — opening another collapses the previous one automatically.

Changes are saved automatically as you type (most fields) or when you confirm a picker selection.

Copying a Section

Right-click a section in the list (or its tile on the grid) and choose Copy. A new section is inserted immediately below the original, pre-filled with:

  • The same course
  • An auto-incremented section code (e.g. AB1 → AB2), if derivable
  • The same campus (if inferred from a section prefix)

If the derived code is already taken, a notice appears and the code is left blank for you to enter manually.

Section Code Uniqueness

Within a semester, no two sections of the same course may share the same section code (comparison is case-insensitive). TermPoint enforces this at validation time and shows an inline error if there is a conflict.

Deleting a Section

Open the section editor and click the Delete button at the bottom of the form. You will be asked to confirm before the section is permanently removed.

Deletion cannot be undone. If you need to temporarily remove a section from the view, consider changing its meeting type to a filtered-out value instead.
